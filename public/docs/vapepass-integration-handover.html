Integration Guide: Install the VapePass AI Assistant

Share this document with your in-house developer or web agency. It covers everything required to install the VapePass chatbot on your store website.

Document version 1.0.0 Estimated time: 15–30 minutes HTML / JS embed

1. Overview

VapePass provides an AI Flavor Sommelier chatbot that recommends products from your live inventory. Installation is a single embed script placed on your storefront. The script is domain-locked to your authorized hostname for security.

Prerequisite: The store owner must complete AI Assistant setup in the VapePass dashboard (connect product page URL, sync inventory, and copy the store-specific embed code) before going live.

2. What you need from the store owner

  • Active VapePass subscription
  • Store-specific embed script from Dashboard or AI Assistant
  • Authorized website domain (hostname must match the account setting)
  • Access to edit the storefront theme / HTML (typically before </body>)

3. Installation steps

  1. Log in to the VapePass dashboard as the store owner.
  2. Open AI Assistant and confirm the product page URL is connected and inventory sync status is successful.
  3. Copy the embed script from the Dashboard or AI Assistant page.
  4. Paste the script into your website template immediately before the closing </body> tag on pages where the assistant should appear (usually all storefront pages).
  5. Publish / deploy the theme changes.
  6. Open the live website and verify the chat widget loads on the authorized domain.
  7. In VapePass, use Finish Setup / Go Live if the assistant is not yet marked live.

Embed snippet (shape)

Your production snippet is store-specific. It will look similar to: 

<script
  src="https://YOUR_API_HOST/widget.js"
  data-store-id="YOUR_STORE_ID"
  async
></script>
Always use the exact script from the customer’s VapePass dashboard. Do not invent or reuse another store’s data-store-id. The script loads the same chat UI used on the VapePass site (via a transparent iframe to your CLIENT_URL /embed page). For local testing only, you may add data-skip-site-age="true" to bypass the host-site age gate.

4. Platform tips

Shopify

  • Online Store → Themes → Edit code → theme.liquid (or relevant layout)
  • Paste the script before </body>
  • Save and preview on the live domain

WordPress / WooCommerce

  • Appearance → Theme File Editor → footer template, or use a “Header/Footer Scripts” plugin
  • Paste before </body> and clear caches

Custom / headless storefronts

  • Add the script in the root HTML layout used for customer-facing pages
  • Ensure it is not stripped by CSP without allowing the VapePass script origin

5. Domain & security notes

  • The widget only works on the store’s authorized hostname.
  • If the site uses www vs non-www, confirm the hostname matches dashboard settings.
  • Optional age-gate integration: set data-require-site-age="true" on the script and ensure your site sets localStorage.vapepass_site_age_verified = "true" (or age_verified) after age verification.

6. Verification checklist

  • Widget appears on desktop and mobile
  • Chat opens and returns product recommendations
  • Inventory in dashboard matches storefront catalog expectations
  • Assistant status in VapePass shows as live
  • Widget does not load on unauthorized domains

7. Troubleshooting

  • Widget missing: Confirm script placement, publish status, and ad/script blockers.
  • Wrong domain error: Update authorized hostname in VapePass settings / assistant setup.
  • No recommendations: Re-run inventory sync and confirm recommendable products exist.
  • Cached theme: Hard-refresh or purge CDN / Shopify cache after deploy.

8. Need help?

Store owners can request free setup assistance from the VapePass dashboard welcome flow. Your team can also ask the store owner to share dashboard access or a staging URL for a joint install session during business hours.
